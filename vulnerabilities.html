Research Findings: Critical Vulnerabilities

Below is a breakdown of the specific security flaws I analyzed during my research into Qualys and Symantec tools.

1. Qualys (Vulnerability Management Platform)

CVE-2025-43079: The "Unchecked Path" Flaw

What it is: The software’s uninstallation script was careless about where it looked for system commands. It’s like a delivery driver accepting a package from a stranger on the street instead of going to the warehouse.

The Damage: A hacker with low-level access can trick the system into running their malicious code, instantly gaining "Root" (Super-Admin) privileges and total control over the server.

CVE-2023-28140: Windows DLL Hijacking

What it is: On Windows computers, the Qualys agent looked for helper files (DLLs) in an unsafe order. I found that an attacker could plant a fake, virus-laden file that the agent would pick up by mistake.

The Damage: The malicious file runs with "System" privileges, allowing an attacker to install backdoors or steal data without being detected.

CVE-2023-6147 & CVE-2023-6149: Jenkins Plugin Weakness

What it is: The plugins connecting Qualys to Jenkins (a software building tool) failed to filter incoming data properly. They accepted "confusing" XML files that they shouldn't have.

The Damage: Attackers could trick the server into revealing secret files, passwords, or internal network details.

2. Symantec (Data Loss Prevention - DLP)

CVE-2024-23617: Critical Buffer Overflow

What it is: This is a flaw where the system tries to read a document that is deliberately "too big" for its memory. Imagine pouring a gallon of water into a cup; the spill goes everywhere. In software, this "spill" allows a hacker to overwrite the system's instructions.

The Damage: A hacker can take full control of the DLP server—the very machine meant to protect data. They could then turn off security rules, delete logs to hide their tracks, and steal the sensitive data of millions of users.

OpenSSL "False Alarms" (Research Finding)

What it is: Security scanners flagged 8 different "Critical" issues with the encryption technology (OpenSSL) used inside Symantec agents.

The Outcome: My research proved these were false positives. The software contained the vulnerable code but never actually used it. This finding saved the IT team from wasting weeks on unnecessary updates.
